6.3 Route Maps
6.3.1 Description
A route map defines a set of rules or conditions. This route map is invoked by route redistribution and policy- based routing for defining the required criteria.
6.3.2 Command Line Configuration
Commands for defining route maps are specified in the router configuration file. The name of the router configuration file is specified in the scenario configuration (.config) file.

6.3.2.1 Format of Route Map Commands in Router Configuration File
Route maps are specified in the router configuration file using the following format:
[<Node Specification>]
[<Interface Specification>]
<Add-route-map Command>
[<Match Commands>]
<Action Commands>
These elements are described in Table 6-12.
Note: 
All entries in the router configuration file are case insensitive.
TABLE 6-12.  Route Map Specification 
Element
Description
<Node Specification>
The node specification identifies the node for which router configuration parameters (including route maps) are being defined and has the following format:
NODE-IDENTIFER <node-ID>
where
<node-ID> Node ID.
The node specification is optional and if it is not included, then the router configuration parameters apply to all nodes.
<Interface Specification>
The interface specification identifies the interface for which router configuration parameters (including route maps) are being defined and has the following format:
INTERFACE <interface-number>
where
<interface-number> Interface number.
The interface specification is optional and if it is not included, then the router configuration parameters apply to all interfaces.
<Add-route-map Command>
A route map is added by using the following command:
ROUTE-MAP <map-tag> [<filter>] [<seq-num>]
where
<map-tag> Name associated with the route map.
<filter> This can be PERMIT or DENY. The filter specification is optional and if it is not included, then the default value is PERMIT.
If the filter is PERMIT, and criteria specified by the match commands are met, then the packet is processed as specified by the action commands.
If the filter is PERMIT, and criteria specified by the match commands are not met, the next route map with the same map tag is tested. If a route passes none of the match criteria for the set of route maps sharing the same name, then the normal forwarding algorithm is used.
If the filter is PERMIT, and criteria specified by the match commands are met, then the normal forwarding algorithm is used. No further route maps sharing the same map tag name are examined.
<seq-num> Sequence number of the route map in the list of route maps configured with the same name. The sequence number specification is optional.
<Match Commands>
Match commands can be specified to match packet lengths or IP addresses.
The command to match the packet length has the following format:
MATCH LENGTH <min> <max>
where
<min> Minimum packet length.
<max> Maximum packet length.
The command to match the IP address has the following format:
MATCH IP ADDRESS <list-of-ACLs>
where
<list-of-ACLs> Non-empty list of access list numbers and/or access list names. See Section 6.5 for details of router access lists.
Note: The match commands use the Cisco syntax. None, one, or both match commands can be specified for a route map. If a match command is not included, then the route map applies to all packets.
<Action Commands>
The action commands specify the action(s) to be performed when a packet matches the criteria specified by the match command(s). One or more of the following commands can be specified.
Note: The action commands use the Cisco syntax.
To set the precedence value in the IP header, use the following command:
SET IP PRECEDENCE <precedence>
where
<precedence> Precedence number or precedence name to which the precedence filed of the IP header should be set.
To set the next hop to which to route the packet, use the following command:
SET IP NEXT-HOP <address-list>
where
<address-list> Non-empty list of IP addresses.
The next hop must be adjacent to the node.
To set the output interface for the packet, use the following command:
SET INTERFACE <interface-list>
where
<interface-list> Non-empty list of interface type/ interface name pairs.
To set the default next hop to which to route the packet when there is no explicit route to the destination, use the following command:
SET IP DEFAULT NEXT-HOP <address-list>
where
<address-list> Non-empty list of IP addresses.
To set the default output interface for the packet when there is no explicit route to the destination, use the following command:
SET DEFAULT INTERFACE <interface-list>
where
<interface-list> Non-empty list of interface type/ interface name pairs.
Examples of Route Maps in Router Configuration File
The following lines show examples of route map configuration in the router configuration file.
 
ROUTE-MAP SRC-ROUTE-3IF0 PERMIT 1
MATCH LENGTH 500 700
SET IP NEXT-HOP 192.168.3.2
#
 
ROUTE-MAP LOAD-SHARE PERMIT 1
MATCH IP ADDRESS 106
SET IP NEXT-HOP 192.168.2.3
6.3.3 GUI Configuration
This section describes how to configure route map in GUI.
 
Configuring Route Map Parameters
To configure the Route Map parameters, perform the following steps:
1.
Go to Default Device Properties Editor > Node Configuration > Router Properties.
2.
Set Router Configuration File to the name of the router configuration file. The format of the router configuration file is described in Section 6.3.2.1.
