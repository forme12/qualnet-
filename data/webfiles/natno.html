NatNo is the preferred mode for running IPNE. NatNo is the easiest mode to configure and will work with all typical IPNE scenarios.   IP packets are sniffed/forwarded between pairs of operational hosts. This means that IPNE will emulate any IP traffic sent from an operational host to another operational hostThe followings are the network configuration of the example ($QUALNET_HOME/scenario/ipne/NatNo).

 

 

* Configuration

 

# Configure IPNE to run in NatNo mode

NAT            NO

TRUE-EMULATION NO

 

# Set a receive delay of 10US

RECEIVE-DELAY  10US

 

# Print out the maximum amount of descriptive information

PRINT-DEVICES       YES

PRINT-PACKET-LOG YES

DEBUG            YES

PRINT-STATISTICS YES

 

# Add one sniffing device.  DEVICE-1 corresponds to the first device

# written in the "PRINT-DEVICES" section

NUM-DEVICES 1            --------------------------------------------------------------  (1)

DEVICE      DEVICE-1     -------------------------------------------------------------- (2)

 

# Configure 2 virtual nodes 192.168.0.45 and 192.168.0.46.  Both

# will use DEVICE-1.  Nodes with IP addresses 192.168.0.45 and

# 192.168.0.46 must exist in the simulator.

NUM-VIRTUAL-NODES 2      ----------------------------------------------------------------------  (3)

VIRTUAL-NODE      REAL-ADDRESS 192.168.0.45 DEVICE DEVICE-1  ------------  (4)

VIRTUAL-NODE      REAL-ADDRESS 192.168.0.46 DEVICE DEVICE-1  ------------  (5)

 

In this simple example, there are no virtual nodes available, but virtual nodes and operational nodes.  There is a physical interface at QualNet machine ((1) and (2)).  And 2 virtual nodes are configured (3) with IP addresses of 192.168.0.45 and 192.168.0.46 ((4) and (5)).  These two nodes are connected to interface DEVICE-1 (5).   If there are multiple physical interfaces at QualNet machine, (1) and (2) should reflect it accordingly.  And each virtual nodes should be configured that each operational node is attached.  Note that each virtual node is the corresponding node of operational node.  Therefore, each virtual node has the same IP address as operational nodes (NatNo mode).  In case of NatYes, IP address of operational node is different from that of virtual node.  We will explain with NatYes mode.  For more information regarding IPNE configuration parameters, refer to the IPNE user’s guide (QualNet-4.0-ModelLibrary-NetworkEmulationInterface.doc).

 

* Operation

 

QualNet is ready to run, and IPNE will capture a packet from operational nodes (192.168.0.45 and 192.168.0.46).  The PCAP filter will be printed right after QualNet runs for verification.

Configure IP address of QualNet as 192.168.0.100, and ping operational nodes (192.168.0.45 and 192.168.0.46)  from QualNet to verify the connection.

 

At QualNet machine

 

> ifconfig eth0 192.168.0.100    for linux

> ping 192.168.0.45

> ping 192.168.0.46

 

At operational nodes, packets should be routed to QualNet.  Otherwise, packets can be delivered from one operational node to the other since they are on the same network.

 

At operational node (192.168.0.45)

> route add –host 192.168.0.46 gw 192.168.0.100                                        for linux

> route add 192.168.0.46 mask 255.255.255.255 192.168.0.100        for windows

 

At operational node (192.168.0.46)

> route add –host 192.168.0.45 gw 192.168.0.100                                          for linux

> route add 192.168.0.45 mask 255.255.255.255 192.168.0.100          for windows

 

Now, you should be to ping operational node from the other operational node.  For debugging purpose, check the debug information on command prompt.  There should be the same number of packets coming in and out.
