In TrueEmulation mode, IPNE creates a packet and inject it in network layer.  Therefore, network layer protocol such as a routing protocol can be injected into the QualNet scenario.  Also, with TrueEmulation, users can access the simulated network.  For example, users can do the PING or TRACEROUTE to the simulated nodes.  NatNo and NatYes are used to deliver the application packets between operational nodes, and TrueEmulation is useful to verify the interaction of real-world packets and simulated network (routing protocols are good example).The followings are the network configuration of the example ($QUALNET_HOME/scenarios/ipne/TrueEmulation).

 

 * Configuration

 

# Configure IPNE to run in TrueEmulation mode

NAT            NO                                     ----------------------------------------------------  (1)

TRUE-EMULATION YES                     ----------------------------------------------------  (2)

 

# Set a receive delay of 10US

RECEIVE-DELAY  10US

 

# Print out the maximum amount of descriptive information

PRINT-DEVICES    YES

PRINT-PACKET-LOG YES

DEBUG            YES

PRINT-STATISTICS YES

 

# Add one sniffing device.  DEVICE-1 corresponds to the first device

# written in the "PRINT-DEVICES" section

NUM-DEVICES    1                                         ----------------------------------------------------  (3)

DEVICE         DEVICE-1                                 ----------------------------------------------------  (4)

 

# Configure 2 virtual nodes 192.168.0.45 and 192.168.0.46.  Both

# will use DEVICE-1.  Nodes with IP addresses 192.168.0.45 and

# 192.168.0.46 must exist in the simulator.  The mac address for each

# virtual node must match the mac address of each operational host

 

NUM-VIRTUAL-NODES 2                      ----------------------------------------------------  (5)

 

VIRTUAL-NODE      REAL-ADDRESS 192.168.0.45 MAC-ADDRESS 00:02:3f:94:98:cd DEVICE  DEVICE-1                      ----------------------------------------------------  (6)

 

VIRTUAL-NODE      REAL-ADDRESS 192.168.0.46 MAC-ADDRESS 00:08:74:3a:15:29 DEVICE  DEVICE-1                       ----------------------------------------------------  (7)

 

# Enable the OSPF interface

OSPF              YES

NUM-OSPF-NODES    1

OSPF-NODE-ADDRESS 192.168.0.46

 

The configuration above is for the True emulation mode ((1) and (2)).  In the scenario, there is a physical interface at QualNet (3) with the name of DEVICE-1 (4).  2 operational nodes are configured (5) with IP address of 192.168.0.45 and 192.168.0.46.  Since IPNE needs to handle the ARP request, MAC address is required for each operational node ((6) and (7)). 

Note that a routing protocol is running on the operational node (192.168.0.46) as well as simulated nodes.  The routing protocol should be running on operational node since the routing table should be constructed on operational node and this is used to forward a packet into QualNet.

 

* Operation

 

Packets from operational nodes should be forwarded to QualNet machine so that IPNE can capture.  

 

At QualNet machine

 

> ifconfig eth0 192.168.0.100    for linux

> ping 192.168.0.45

> ping 192.168.0.46

 

At operational node (192.168.0.45)

> route add –host 192.168.0.46 gw 192.168.0.100                                        for linux

> route add 192.168.0.46 mask 255.255.255.255 192.168.0.100        for windows

 

At operational node (192.168.0.46)

> route add –host 192.168.0.45 gw 192.168.0.100                                          for linux

> route add 192.168.0.45 mask 255.255.255.255 192.168.0.100          for windows
