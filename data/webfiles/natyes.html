The difference between NatYes and NatNo is the IP address of virtual nodes and operational nodes.  In NatYes mode, the IP address of virtual node is different from operational node.  Therefore, this mode is useful when you have to assign different IP address to QualNet scenario from the real network.  The followings are the network configuration of the example ($QUALNET_HOME/scenario/ipne/NatYes).

 

 * Configuration

 

# Configure IPNE to run in NatYes mode

NAT            YES                ------------------------------------------------------------------------  (1)

TRUE-EMULATION NO    ------------------------------------------------------------------------  (2)

 

# Set a receive delay of 10US

RECEIVE-DELAY  10US

 

# Print out the maximum amount of descriptive information

PRINT-DEVICES    YES

DEBUG            YES

PRINT-PACKET-LOG YES

PRINT-STATISTICS YES

 

# Calculate all checksums explicitly

EXPLICITLY-CALCULATE-CHECKSUM YES

 

# Route unsupported protocols (not TCP, UDP or ICMP) directly through

# QualNet

ROUTE-UNKNOWN-PROTOCOLS YES

 

# Add one sniffing device.  DEVICE-1 corresponds to the first device

# written in the "PRINT-DEVICES" section

NUM-DEVICES    1                ----------------------------------------------------------------------  (3)

DEVICE         DEVICE-1        ----------------------------------------------------------------------  (4)

 

# Configure 2 virtual nodes 192.168.0.45 and 192.168.0.46.  Both

# will use DEVICE-1.  Nodes with IP address 100.100.0.45 and 100.100.0.46

# must exist in the simulator.

NUM-VIRTUAL-NODES 2         -------------------------------------------------------------------  (5)

VIRTUAL-NODE      REAL-ADDRESS 192.168.0.45 PROXY-ADDRESS 100.100.0.45 DEVICE DEVICE-1          -------------------------------------------------------------------  (6)

VIRTUAL-NODE      REAL-ADDRESS 192.168.0.46 PROXY-ADDRESS 100.100.0.46 DEVICE DEVICE-1          -------------------------------------------------------------------  (7)

 

The configuration above is for the NatYes mode ((1) and (2)).  In the scenario, there is a physical interface at QualNet (3) with the name of DEVICE-1 (4).  2 virtual nodes are configured (5), and they are connected to the operational nodes with different IP address ((6) and (7)).  The operational node with IP address, 192.168.0.45 is matched with the virtual node with IP 100.100.100.0.45.  And the operational node with IP address, 192.168.0.46 is matched with the virtual node with IP 100.100.100.0.46.  This configuration means that the packets from/to 192.168.0.45 should be treated as the packets are from/to 100.100.0.45 in QualNet.  This mode is useful because you can connect the operational node to any node (any simulated node) in QualNet scenario. 

* Operation

As in NatNo mode, packets from operational nodes should be forwarded to QualNet machine so that IPNE can capture.

 

At QualNet machine

 

> ifconfig eth0 192.168.0.100    for linux

> ping 192.168.0.45

> ping 192.168.0.46

 

At operational node (192.168.0.45)

> route add –host 100.100.0.46 gw 192.168.0.100                                        for linux

> route add 100.100.0.46 mask 255.255.255.255 192.168.0.100        for windows

 

At operational node (192.168.0.46)

> route add –host 100.100.0.45 gw 192.168.0.100                                          for linux

> route add 100.100.0.45 mask 255.255.255.255 192.168.0.100          for windows
